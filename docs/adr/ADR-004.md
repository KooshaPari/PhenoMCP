# ADR-004: PhenoMCP Security Model

## Status

**Accepted**

## Context

MCP enables AI models to execute tools and access resources, creating significant security implications. A robust security model must protect against unauthorized access, data exfiltration, and resource abuse.

Key challenges:

1. **Authorization**: Who can invoke which capabilities
2. **Authentication**: Verifying identity of clients and servers
3. **Sandboxing**: Isolating tool execution
4. **Audit**: Tracking all capability invocations
5. **Secrets Management**: Handling credentials safely
6. **Data Protection**: Preventing unauthorized data access

## Decision

We will implement defense-in-depth security:

### 1. Authentication Architecture

**Rationale**: Multiple authentication options for different scenarios.

**Decision Details**:
- Token-based authentication (JWT, API keys)
- mTLS for service-to-service
- OAuth 2.0/OIDC for user authentication
- HMAC request signing
- Anonymous mode for local development only
- Token refresh without reconnection

### 2. Authorization Framework

**Rationale**: Fine-grained access control for capabilities.

**Decision Details**:
- Role-based access control (RBAC)
- Attribute-based access control (ABAC) for complex cases
- Capability-level permissions
- Resource-level permissions with URI patterns
- Time-based access restrictions
- Audit logging of all authorization decisions

### 3. Tool Execution Sandbox

**Rationale**: Untrusted tools must be isolated.

**Decision Details**:
- WASM sandbox for portable isolation
- Process isolation for native tools
- Container isolation for high-risk tools
- Network egress filtering
- Filesystem access controls
- Resource quotas (CPU, memory, time)

### 4. Secrets Management

**Rationale**: Tools often need credentials without exposure.

**Decision Details**:
- Integration with secret stores (HashiCorp Vault, AWS Secrets Manager)
- Environment variable injection
- Credential binding at tool registration
- No secrets in tool schemas or logs
- Automatic secret rotation support
- Audit trail for secret access

### 5. Audit and Compliance

**Rationale**: Security requires visibility.

**Decision Details**:
- Structured logging of all requests
- Tamper-evident audit logs
- Real-time alerting on suspicious patterns
- Data retention policies
- Compliance reporting (SOC2, GDPR)
- Export to SIEM systems

## Consequences

### Positive

1. **Security**: Multiple defense layers
2. **Compliance**: Meets regulatory requirements
3. **Trust**: Auditable operations build confidence
4. **Flexibility**: Multiple auth options for different needs
5. **Isolation**: Sandboxing prevents lateral movement
6. **Visibility**: Comprehensive audit trail

### Negative

1. **Complexity**: Security features add implementation complexity
2. **Performance**: Auth checks and sandboxing add overhead
3. **Configuration**: Many security options to configure
4. **Learning Curve**: Security model requires understanding
5. **Maintenance**: Security updates require attention

### Neutral

1. **Operational**: Security operations expertise needed
2. **Cost**: Security infrastructure has cost
3. **User Friction**: Security measures may slow development

## Alternatives

### Alternative 1: Trust Boundary

**Considered**: Assume MCP runs in trusted environment, no auth.

**Rejected Because**:
- Dangerous for production use
- No audit trail
- Cannot support multi-tenant scenarios
- Violates security best practices

### Alternative 2: Network Isolation Only

**Considered**: Rely on network policies for security.

**Rejected Because**:
- Insufficient for application-level security
- No user authentication
- Bypass possible
- No audit logging

### Alternative 3: External Proxy

**Considered**: Handle auth in reverse proxy.

**Rejected Because**:
- Application-level auth still needed
- Adds infrastructure complexity
- Cannot do fine-grained capability auth

### Alternative 4: OS-Level Security

**Considered**: Rely on OS permissions and containers.

**Rejected Because**:
- Not portable across platforms
- Complex to configure
- No audit trail
